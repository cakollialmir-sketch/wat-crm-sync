# Workflow: CRM Call Sync

## Objective
Automatically sync every call outcome into GoHighLevel (GHL) without manual data entry. Every time you end a call in Call Tools, a prospect books on Calendly, or a Zoom demo finishes — GHL is updated instantly.

---

## System Map

| Event Source | Trigger | Webhook Endpoint | GHL Actions |
|---|---|---|---|
| Call Tools | Disposition set (any outcome) | `/webhook/calltools` | Upsert contact → move pipeline stage → create task/appointment/note |
| Calendly | Prospect self-books | `/webhook/calendly` | Upsert contact → create appointment → move to Appointment Set → fire confirmation workflow |
| Zoom | Meeting ends | `/webhook/zoom` | Find contact by host email → add demo note → move to Show |
| Manual (cell phone) | You log it via CLI | `tools/log_call.py` | Same as Call Tools |

## Required Environment Variables

Add all of these to `.env` (and to Render dashboard env vars after deploying).

| Variable | Purpose | Where to Get It |
|---|---|---|
| `GHL_API_TOKEN` | Authenticates all GHL API calls | GHL → Settings → Integrations → Private Integrations → Create → copy token (starts with `pit_`) |
| `GHL_LOCATION_ID` | Scopes all data to your GHL location | GHL → Settings → Business Info → scroll down → "Location ID" |
| `GHL_PIPELINE_ID` | Which pipeline to create opportunities in | Run the pipeline API query below |
| `GHL_CALENDAR_ID` | Which calendar to book appointments on | GHL → Calendars → Settings → copy calendar ID from URL |
| `GHL_CONFIRMATION_WORKFLOW_ID` | GHL automation to fire on booking (sends SMS/email confirmation) | GHL → Automation → your workflow → copy ID from URL |
| `GHL_STAGE_NO_ANSWER` | Pipeline stage ID | Run the pipeline API query below |
| `GHL_STAGE_VOICEMAIL` | Pipeline stage ID | Same |
| `GHL_STAGE_CALLBACK` | Pipeline stage ID | Same |
| `GHL_STAGE_APPT_SET` | Pipeline stage ID | Same |
| `GHL_STAGE_SHOW` | Pipeline stage ID | Same |
| `GHL_STAGE_NOT_INTERESTED` | Pipeline stage ID | Same |
| `CALLTOOLS_WEBHOOK_SECRET` | Verifies Call Tools webhook authenticity | You choose — set same string in Call Tools webhook config |
| `CALENDLY_WEBHOOK_SECRET` | Verifies Calendly webhook authenticity | You choose — set same string in Calendly webhook config |
| `ZOOM_WEBHOOK_SECRET` | Verifies Zoom webhook authenticity | Generated by Zoom when creating Webhook Only App |
| `CALENDLY_API_TOKEN` | Fetches invitee phone from Calendly | app.calendly.com → Integrations → API & Webhooks → Personal Access Token |

**Get pipeline + stage IDs:**
```bash
source .env
curl -s -H "Authorization: Bearer $GHL_API_TOKEN" \
  "https://services.leadconnectorhq.com/opportunities/pipelines?locationId=$GHL_LOCATION_ID" \
  | python3 -m json.tool | grep -E '"id"|"name"'
```
Copy the pipeline `id` → `GHL_PIPELINE_ID`. Copy each stage `id` → the matching `GHL_STAGE_*` var.

## Pipeline Stage Map

Create these stages in GHL → Opportunities → Pipelines (in this order):

| Stage Name | `GHL_STAGE_*` var | Triggered by |
|---|---|---|
| New Lead | _(not mapped)_ | Manual |
| Called - No Answer | `GHL_STAGE_NO_ANSWER` | Call Tools: `no_answer` |
| Called - Voicemail | `GHL_STAGE_VOICEMAIL` | Call Tools: `voicemail` |
| Follow Up Scheduled | `GHL_STAGE_CALLBACK` | Call Tools: `callback` + task created |
| Appointment Set | `GHL_STAGE_APPT_SET` | Call Tools: `meeting_booked` or Calendly booking |
| Show | `GHL_STAGE_SHOW` | Zoom: `meeting.ended` |
| Closed Won | _(not mapped)_ | Manual |
| Not Interested | `GHL_STAGE_NOT_INTERESTED` | Call Tools: `not_interested` |

## Setup Checklist (One-Time)

### 1. Install new dependencies
```bash
cd /Users/almircakolli/AnyToResource
pip install fastapi==0.111.0 "uvicorn[standard]==0.30.1" pydantic==2.7.1
```

### 2. Create pipeline stages in GHL
GHL → Opportunities → Pipelines → Create or edit pipeline → add all stages from the table above.
Run the API query to get stage IDs, then fill in all `GHL_STAGE_*` vars in `.env`.

### 3. Deploy webhook server to Render
1. Push this repo to GitHub (or create a new repo: `git init && git add . && git commit -m "init"`)
2. Go to render.com → New → Web Service → Connect GitHub repo
3. Render auto-detects `render.yaml` — click "Apply"
4. In Render dashboard → Environment → add all env vars from `.env` (the `sync: false` ones)
5. Deploy — your webhook base URL: `https://wat-crm-sync.onrender.com`

### 4. Set up UptimeRobot (keeps Render free tier awake)
1. uptimerobot.com → Add Monitor → HTTP(s)
2. URL: `https://wat-crm-sync.onrender.com/health`
3. Monitoring interval: 5 minutes
4. Free tier is sufficient

### 5. Register webhooks in Call Tools
- Call Tools Manager → Settings → Integrations/Webhooks → Add Webhook
- URL: `https://wat-crm-sync.onrender.com/webhook/calltools`
- Events: `Call Disposition Created`, `Call Ended`
- Secret/token: paste your `CALLTOOLS_WEBHOOK_SECRET` value
- Signature header: `X-CallTools-Signature`

### 6. Register webhooks in Calendly
- calendly.com → Integrations → API & Webhooks → Webhook Subscriptions → New Webhook
- URL: `https://wat-crm-sync.onrender.com/webhook/calendly`
- Events: `invitee.created`
- Also grab a Personal Access Token and set as `CALENDLY_API_TOKEN`

### 7. Register webhooks in Zoom
- marketplace.zoom.us → Develop → Build App → Webhook Only App (or Feature)
- Add event subscription: `meeting.ended`
- Notification endpoint URL: `https://wat-crm-sync.onrender.com/webhook/zoom`
- Secret token: your `ZOOM_WEBHOOK_SECRET`
- Save — Zoom sends a validation challenge automatically handled by the server

### 8. Verify (see Verification section below)

## Troubleshooting

**Contact not created in GHL**
- Check `GHL_API_TOKEN` is set and starts with `pit_`
- Check `GHL_LOCATION_ID` matches your sub-account
- Look at Render logs: Render dashboard → your service → Logs

**Pipeline stage not moving**
- Verify the `GHL_STAGE_*` IDs are correct (run the pipeline API query)
- Confirm `GHL_PIPELINE_ID` is set

**Webhook signature failing (401)**
- Make sure the secret in `.env` matches exactly what you entered in the platform's webhook config
- Call Tools may not send a signature on test events — remove the secret temporarily to test, then re-add

**Render service sleeping**
- Set up UptimeRobot as described in Setup step 4
- Or upgrade to Render Starter ($7/month) to eliminate sleep entirely

**Zoom host email doesn't match prospect**
- The Zoom webhook uses the meeting host email to find the GHL contact
- Workaround: include the prospect's email in the meeting topic when booking (e.g. "Demo — prospect@email.com") and add parsing logic in `webhook_server.py`

**Calendly phone not captured**
- Confirm your Calendly event type has a phone number question
- Confirm `CALENDLY_API_TOKEN` is set
- Check server logs for "Could not fetch Calendly invitee phone" warning

**Rate limit errors (429)**
- tenacity retries these automatically with exponential backoff
- If persistent: add `time.sleep(0.1)` between rapid sequential calls

## Edge Cases

- **Duplicate calls to same prospect:** `upsert_opportunity` searches for an existing open opportunity first and patches it — no duplicates
- **Timezone:** Always store datetimes as ISO 8601 with timezone (e.g. `2026-05-29T14:00:00Z`) — GHL API requires it
- **Missing phone on Calendly:** Some prospects skip the phone field — contact is still created with email only; phone can be added manually in GHL
- **Call Tools disposition labels:** The server normalizes common variants (e.g. `"no-answer"`, `"No Answer"`, `"appointment_set"`) — add new variants to `_DISP_NORMALIZE` in `ghl_client.py` if your dialer uses different labels
